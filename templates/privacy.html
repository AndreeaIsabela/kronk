Privacy Policy

Last updated: 2026-06-16 (rev. 2)

This policy explains what data the Kronk Discord bot ("the Bot") collects, why, and how long it is kept.

Data We Collect

Pet Buff Timers (/pet-buff)

When a pet buff timer is started, the following is stored per player:

FieldPurpose
Discord user IDIdentifies whose timer it is
Display nameUsed in timer log recovery
Timer expiry timestampSchedules warning and deactivation messages
Discord channel IDDetermines where notifications are posted
Discord server (guild) IDIsolates data per server

Rally Leaders (/set-rally-leaders)

When a rally leaders list is saved, the following is stored per server:

FieldPurpose
Discord user IDsIdentifies the rally leaders
Display namesUsed for crash recovery logging
Discord server (guild) IDIsolates the list per server

Scheduled Events (/set-event)

When an event is scheduled, the following is stored per event:

FieldPurpose
Event nameIdentifies and uniquely keys the event within the server
Discord server (guild) IDIsolates data per server
Discord channel IDDetermines where the notification is posted
Setter user IDRecords who created the event
Scheduled timestampDetermines when to fire the notification
FrequencyControls recurrence (once / daily / bi-daily / weekly / monthly)
Custom messageText included in the notification (if provided)
Player Discord user IDsDetermines who is mentioned in the notification

How Data Is Used

All collected data is used exclusively to operate the Bot's timer, notification, and event scheduling features within your Discord server. The /list-events command reads and displays stored event data only to the server member who invoked it. No data is ever sold, shared with third parties, or used for any other purpose.

Where Data Is Stored

Data is stored in a MongoDB database hosted on MongoDB Atlas (cloud). It is not stored locally on any personal device.

Retention

  • Timer data is deleted automatically when the timer expires (within 2 hours). If the Bot is offline when a timer expires, the data is deleted on next startup.
  • Rally leaders data is retained until overwritten by a new /set-rally-leaders command or until the Bot is removed from the server.
  • Event data for one-time events is deleted automatically after the notification fires. Recurring event data is retained until the event is cancelled with /cancel-event or the Bot is removed from the server.

Your Rights

You may request deletion of your data at any time by contacting a server administrator. Timer data can also be cleared immediately using /pet-buff-cancel.

Contact

For privacy questions or data deletion requests, reach out to a server administrator.
